## Slice Rationale

The left rail already owns workflow, attention, worker, and planning navigation. Adding the running section there avoids duplicating the Queue or Workers surfaces.

## Non-goals

Do not change worker action persistence, task execution states, queue ordering, command palette behavior, terminal drawer behavior, or right-side inspector behavior.

## Architecture

### Before

`App` derived running task entries for existing surfaces, but `LeftStatusColumn` only rendered navigation and counts.

### After

`App` passes running entries and task selection into `LeftStatusColumn`, which renders a compact expanded-sidebar list while preserving collapsed navigation semantics.
