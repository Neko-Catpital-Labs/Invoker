# INV-63 Experiment Brief

## Goal
Establish deterministic experiment proof for INV-63 so architecture choices are evidence-backed and reviewable.

## Files under test
- `skills/plan-to-invoker/SKILL.md` -- canonical skill policy and routing surface.
- `.cursor/skills/plan-to-invoker/SKILL.md` -- Cursor-installed mirror, generated by `scripts/setup-agent-skills.sh`.
- `skills/plan-to-invoker/scripts/skill-doctor.sh` -- deterministic validation gate that aggregates extract/generate/validate/lint/parse checks.

## Alternative considerations
- **Selected:** canonical skill at `skills/plan-to-invoker/SKILL.md` driving `skill-doctor.sh`. One authoritative policy, one validation entry point.
- **Alternative A:** treat `.cursor/skills/plan-to-invoker/SKILL.md` as authoritative. Rejected because it is a downstream artifact that drifts unless the setup script is rerun.
- **Alternative B:** hand-rolled per-plan validators replacing `skill-doctor.sh`. Rejected because it loses atomicity, policy-coverage, and parse-results aggregation, and forces reviewers to remember which individual script to call.

## Deterministic commands, expected outputs, verdicts, thresholds

| # | Deterministic command | Expected output | Verdict rule | Threshold |
|---|-----------------------|-----------------|--------------|-----------|
| 1 | `test -s skills/plan-to-invoker/SKILL.md` | exit 0 | PASS iff exit 0 | size > 0 bytes |
| 2 | `test -s .cursor/skills/plan-to-invoker/SKILL.md` | exit 0 | PASS iff exit 0 | size > 0 bytes |
| 3 | `test -s skills/plan-to-invoker/scripts/skill-doctor.sh` | exit 0 | PASS iff exit 0 | size > 0 bytes |
| 4 | `grep -c '^## ' docs/context/inv-63/experiment-brief.md` | integer >= 6 on stdout | PASS iff stdout >= 6 | at least 6 top-level sections |
| 5 | `grep -E 'Verdict\|Threshold' docs/context/inv-63/experiment-brief.md` | non-empty stdout, exit 0 | PASS iff non-empty | verdict + threshold language present |

## Verdict rubric
- **PASS:** all five commands above exit 0 with stdout meeting the thresholds.
- **FAIL:** any command exits non-zero, or command 4 returns a value below 6, or command 5 returns empty stdout.
- **INCONCLUSIVE:** a command runs but produces ambiguous output (for example, a partial file write); rerun `bash skills/plan-to-invoker/scripts/skill-doctor.sh <plan>` for the authoritative gate.

## Threshold notes
- File presence threshold uses POSIX `test -s` so a zero-byte file fails.
- Section threshold is `>= 6` so this brief retains its required structure even if minor sections are added later.
- The authoritative deterministic gate remains `bash skills/plan-to-invoker/scripts/skill-doctor.sh <plan>`; this brief documents the smoke surface, not a replacement for that gate.

## Reviewability
This brief is the single artifact a reviewer reads to judge whether the INV-63 architecture choice is evidence-backed: it names the files under test, the competing design, and the deterministic commands that produce a PASS/FAIL verdict.
